ℹ️ About This Project

Ensemble Learning Model for Predicting Digital Game Success

📚 Project Overview

This system is a thesis project developed at FEU Institute of Technology under the Bachelor of Science in Computer Science with Specialization in Data Science program.

The research applies ensemble learning techniques (Random Forest, Gradient Boosting, XGBoost) combined through stacking to predict the commercial success of digital games on the Steam platform using pre-launch features only.

🔬 Methodology

Data Source

The model was trained on 10,000 games collected from Steam Web API and SteamSpy, focusing exclusively on pre-launch attributes (no post-launch reviews or playtime data).

Target Variable

Games are classified into 6 ordinal owner tiers based on estimated total owners:

  • Class 0: ≤10K owners (Common Indie)
  • Class 1: 10K–35K owners (Niche)
  • Class 2: 35K–75K owners (Growing)
  • Class 3: 75K–150K owners (Established)
  • Class 4: 150K–350K owners (Popular)
  • Class 5: ≥750K owners (Breakout Hit)

Note: Classes 5-8 from the original 9-class scheme were merged into Class 5 due to sparse samples.

Ensemble Architecture

The system employs a stacked ensemble approach:

  1. Base Models: Random Forest, Gradient Boosting, XGBoost (trained independently)
  2. Meta-Learner: XGBoost Classifier (combines base model probabilities)

This two-level architecture leverages the complementary strengths of each algorithm:

  • Random Forest: Robust feature importance, handles non-linearity
  • Gradient Boosting: Sequential error correction, reduces bias
  • XGBoost: Regularization, handles missing data, captures complex patterns

Feature Engineering

From raw Steam data, 48 pre-launch features were extracted and engineered:

  • Raw Features (41): Pricing, release month, genre, platform, languages, screenshots, trailers, developer/publisher info, Steam features, tags, DLC
  • Composite Scores (7): Store page quality, marketing investment, platform reach, publisher backing, localization, Steam integration, mature content flag

Evaluation Metrics

Model performance was assessed using:

  • Weighted F1 Score: Accounts for class imbalance
  • Macro F1 Score: Evaluates performance on minority classes
  • Accuracy: Overall classification correctness

The ensemble achieved:

  • Weighted F1 = 0.6211
  • Macro F1 = 0.2688
  • Accuracy = 70.85%

🔍 Key Findings

  • Most Influential Features: Store page quality (trailers, screenshots, detailed descriptions), publisher backing, platform reach, and localization support.
  • Genre Effects: Indie games dominate the dataset but show lower average success rates. Action, Strategy, and RPG genres correlate with higher owner tiers.
  • Pricing Strategy: Mid-range pricing ($9.99–$19.99) shows optimal performance. Free-to-play games exhibit bimodal distribution (either very successful or fail quickly).
  • Release Timing: October–December release months correlate with slightly higher success (likely due to holiday sales).
  • Steam Integration: Achievements, trading cards, and Workshop support positively correlate with owner tier, suggesting community engagement features matter.

⚠️ Limitations

  • Platform-Specific: Model is trained exclusively on Steam data. Predictions for other platforms (Epic, GOG, consoles) are not reliable.
  • Temporal Drift: Gaming trends evolve rapidly. The model should be retrained periodically with updated data to maintain accuracy.
  • Data Quality: SteamSpy provides estimated owner ranges (not exact counts), which introduces some noise into the training data.
  • Causation vs Correlation: The model identifies correlations, not causation. A high prediction doesn't guarantee success—game quality is the ultimate factor.
  • Niche Genres: Performance on underrepresented genres (e.g., Racing, Sports) may be less reliable due to limited training samples.
